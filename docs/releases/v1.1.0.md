# mini-agent v1.1.0

Release date: 2026-04-13

`v1.1.0` is the hardening release for `mini-agent`. It closes the v1.1 entry gate and turns the earlier v1.0 baseline into a documented, diagnosable, maintainable runtime release.

## Highlights

### Session grammar and recovery

- added session health states: `valid`, `degraded`, `corrupted`
- added strict vs recover load modes
- added `session show --recover`
- restored tool-result replay and skill activation on resume
- improved corruption reporting with repair notes and repair-report files

### Tool execution contract

- normalized tool result metadata under `meta`
- normalized artifacts under `artifacts[].type`
- added runtime-level argument validation and timeout enforcement
- bounded `read` without whole-file loading
- bounded `bash` output in memory while streaming full logs to artifacts

### Operator diagnostics

- grouped `doctor` output into runtime, provider, skill, session, and smoke-path sections
- added smoke-path configuration fields
- improved OpenAI-compatible provider error categorization

### Maintainer documentation

- added session, tool, model adapter, and skill-writing reference docs
- split field-level reference detail out of the v1.1 delta spec

### First-skill readiness

- selected `intel-bulletin` as the first real-skill target
- added stable fixtures and a readiness integration test

## Scope boundary

This release does not include:

- assistant token streaming
- provider fallback chains
- richer session branching
- subagents
- remote execution / sandbox / MCP runtime
- automatic compaction

Those remain post-v1.1 planning decisions.
