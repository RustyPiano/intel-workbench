# 产品设计 + 界面规格：情报分析工作台

> 面向情报部门的离线智能情报处理与多模态分析系统。
> 本文是产品与界面层的设计蓝图：先定形态、角色、流程与每屏规格，再进入工程实现。
> 配套：运行时见 [runtime-architecture](../explanation/runtime-architecture.md)；
> 已有能力见 [intel-bulletin spec](./intel-bulletin-spec.md)、
> [av-dialogue-insight spec](./av-dialogue-insight-spec.md)。

---

## 1. 概述与背景

### 1.1 是什么

一套**离线运行**的情报分析应用：情报人员把已掌握的原始材料（文档、音频、视频、图片，
可能多语种）汇入系统，系统在本地完成识别、转写、翻译、要素抽取、语义检索与报告生成，
全过程结果**可溯源、可复核、可审计**。

### 1.2 形态定性

**不是聊天机器人，而是以"专题"为中心的工作台。** AI 是嵌在工作流里的参谋助手，
不是产品入口。对话只是工作台中的一个面板。详见 §2、§4。

### 1.3 硬约束（贯穿全篇）

- **离线 / 数据只进不出**：系统不主动联网爬取，只分析用户提供的文件；分析区零外发，
  由基础设施（默认拒绝出网 + 离线环境）强制，不依赖软件自觉。
- **开源模型**：所有模型为开源权重，经统一可插拔的模型层接入（开发期可用云端 API 承载
  的开源模型替身，禁用商业模型）。模型选型与部署不在本文范围。
- **低算力可移植**：同一套软件按"算力档位"配置部署，不写死模型尺寸。
- **全程审计**：关键动作哈希链留痕，保密员可查。

### 1.4 不在本文范围

模型选型/部署/量化、网络爬取与采集、组网与等保测评细则、版式精排。本文只定**产品与界面**。

## 2. 产品定位与目标

### 2.1 解决的实际问题

把一摞异构、可能多语种的原始材料，从"靠人通读、转写、交叉比对几小时到几天"，
压缩为"几分钟得到一份可检索、要素已标注、时间线已排好、条条可溯源的卷宗，并一键出公文"。

### 2.2 目标用户

情报部队官兵——**非技术人员**。习惯按专题/任务办公，重视准确、溯源、规范公文与效率，
处理涉密材料，必须人始终掌控、结论可证。

### 2.3 成功标准（产品级）

1. 一名零培训的作业员能独立走完"建专题 → 汇入 → 分析 → 出报告"全流程。
2. AI 给出的每条结论都能一键点回原始素材出处（溯源覆盖率 100%）。
3. 作业员界面**不出现**任何"模型 / 提示词 / token / Agent"等技术词。
4. 管理员能在后台完成提示词模板、Skill、模型、用户、审计的全部管理，无需改代码。
5. 全流程关键动作均落入可查、防篡改的审计日志。

---

## 3. 用户与角色

系统采用**双界面**：作业面（极简、引导式）与管理面（功能完整）分离。这样既满足"完整
管理系统"的要求，又不让官兵被技术细节劝退。

| 角色 | 进入界面 | 能做 | 看不到 |
| --- | --- | --- | --- |
| **作业员**（官兵） | 专题工作台 | 建专题、汇入素材、看要素、问答分析、起草报告 | 模型/提示词/Skill 等技术配置 |
| **管理员**（技术保障） | 管理后台 | 提示词模板、Skill、模型配置、用户权限管理 | —— |
| **保密员**（审计） | 审计中心（只读） | 查全量审计、导出留存、复核边界事件 | 不能修改业务数据 |

> 角色与"可访问密级"绑定：用户只能进入不高于自身权限密级的专题（§8.14）。

## 4. 设计原则

1. **专题为中心，不是对话为中心**——入口是卷宗，不是聊天框。
2. **一切结论可溯源**——每条 AI 输出都挂出处，可点回原文。这是 AI 在情报场景被信任的前提。
3. **人在环路**——AI 提议、人确认；OCR/转写可校对，报告必须人工复核才能导出。
   AI 是参谋不是决策者。
4. **隐藏技术细节**——作业面只说"识别/转写/翻译/分析"，绝不暴露底层。
5. **密级与离线状态常驻可见**——顶栏永远显示当前密级和"● 离线"，给用户安全体感。
6. **可降级、不黑盒**——任一加工环节失败/低置信，明确提示并给人工兜底入口，不假装成功。

## 5. 核心概念与信息架构

### 5.1 领域模型

```
专题(Case)
 ├─ 素材(Material)*        汇入的文件 + 归一化文本/转写/OCR/译文 + 置信度
 ├─ 要素(Element)*         人物/地点/组织/时间/装备/事件，含"提及"列表(指回素材出处)
 │   └─ 关系(Relation)*    主体—关系—客体，附出处
 ├─ 时间线(Timeline)       事件序列，每事件关联要素与出处
 ├─ 问答记录(Inquiry)*     问题 + 回答 + 引用出处 + 是否加入报告
 └─ 报告(Report)*          公文式情报通报：spec + 渲染产物 + 复核状态

（管理面）
 提示词模板(PromptTemplate)* | Skill* | 模型配置(ModelConfig)* | 用户(User)* | 审计事件(AuditEvent)*
```

### 5.2 关键实体字段（产品视角，非数据库 DDL）

- **专题**：`id, 名称, 密级, 说明, 创建人, 创建/更新时间, 状态(加工中|已完成|已归档), 素材数`
- **素材**：`id, 专题id, 文件名, 类型(文档|音频|视频|图片), 原格式, 大小, 汇入时间,
  加工状态(待加工|加工中|已完成|失败), 语种, 归一化文本, 转写/OCR结果, 置信度, 译文?`
- **要素**：`id, 类型, 名称, 别名[], 提及[{素材id, 位置/时间戳, 原文片段}], 出现次数, 备注`
- **报告**：`id, 专题id, spec(标题/密级/概要/小节/结论/签发/日期), 产物(.md/.docx),
  状态(草稿|已复核|已导出), 复核人`
- **审计事件**：`id, 时间, 用户, 动作, 对象, 结果, 本条哈希, 上一条哈希`（哈希链，§11.3）

### 5.3 工作区落盘布局（与 intel-bulletin 既有约定对齐）

```
cases/<case-id>/
  materials/        原始素材（拷入）
  processed/        归一化文本 / 转写 / OCR / 译文（每素材一份）
  elements.json     抽取出的要素、关系、时间线
  inquiries.jsonl   问答记录（含引用出处）
  report/           报告 spec + 渲染产物(.md, 可选 .docx)
  manifest.json     专题元数据 + 素材清单
  audit.log         本专题哈希链审计（全局审计另有汇总，§8.15）
```

## 6. 核心使用流程

主轴（作业员视角）：

```
建立专题 → 汇入素材 → 自动加工 → 交互分析 → 起草报告 → 复核导出
 (命名定密)  (拖入文件)  (OCR/转写/    (问答+溯源)  (一键公文)   (人工核准
                        翻译/抽要素/                            + 审计留痕)
                        建索引)
```

1. **建立专题**：填名称、选密级、写说明（§8.3）。
2. **汇入素材**：拖入混合格式材料；系统逐件显示"识别/转写/翻译"进度（§8.10）。
3. **自动加工**：完成后自动抽取要素、构建时间线、建立语义检索索引。用户看到的是
   结构化结果，不是模型原始输出。
4. **交互分析**：用大白话提问，回答**带回链到素材出处**；可"加入报告"（§8.7）。
   低置信的转写/识别处有"⚠ 点此校对"入口（§9.2）。
5. **起草报告**：一键生成公文式情报通报草稿（复用 intel-bulletin），可编辑（§8.8）。
6. **复核导出**：必须人工复核后才能导出 .md/.docx；导出动作入审计。

> 失败兜底：任一素材加工失败，不阻塞其余素材；该素材标"失败"并给重试/手动录入入口。

## 7. 全局界面框架

所有界面共用同一外壳：

```
┌─ {应用名}/{当前位置} ───────────[密级: 机密]──[● 离线]──[用户 ▾]──[管理后台]─┐
│  ← 顶栏：左=面包屑导航，右=密级徽标 / 离线状态 / 当前用户 / 管理入口(仅管理员)   │
├──────────────────────────────────────────────────────────────────────────────┤
│                              ← 主内容区                                         │
└──────────────────────────────────────────────────────────────────────────────┘
```

- **密级徽标**：随当前专题/页面动态显示，配色分级（内部/秘密/机密/绝密）。
- **● 离线**：常驻；若检测到任何外发尝试被拦截，转为告警态并提示保密员。
- **管理后台**入口仅对管理员可见。

## 8. 界面规格（逐屏）

每屏含：用途、布局、元素、交互、状态。

### 8.1 登录与角色

- **用途**：本地账号登录，确定角色与可访问密级。
- **元素**：用户名、口令；（可选）二要素。登录后按角色进入：作业员→首页，
  管理员→首页+管理入口，保密员→审计中心。
- **状态**：失败计数与锁定提示；登录事件入审计。

### 8.2 首页 / 专题列表

- **用途**：作业员的主入口，纵览与进入专题。
- **布局**：

```
┌─ 情报分析工作台 ─────────────────[密级:机密]──[● 离线]──[张三 ▾]──[管理后台]─┐
│  我的专题                                       [🔍 搜索]    [ + 新建专题 ]     │
│  ─────────────────────────────────────────────────────────────────────────   │
│  □ 东南沿海动态           机密   素材 23   更新 2小时前   ● 加工中             │
│  □ XX人员关系核查         秘密   素材  8   更新 昨天      ✓ 已完成             │
│  □ 截获通话批次0531       机密   素材 47   更新 3天前     ✓ 已完成             │
│  □ （已归档）某专题       秘密   素材 12   归档 上月       🗄 已归档            │
│  …                                                                            │
└──────────────────────────────────────────────────────────────────────────────┘
```

- **元素**：每行=专题名 / 密级 / 素材数 / 更新时间 / 状态徽标。顶部搜索 + 新建。
- **交互**：点击行→进入工作台；右键/悬浮→重命名、归档、删除（按权限）。
  仅显示用户有权访问密级的专题。
- **状态**：空态（"还没有专题，点击新建开始"）；加工中专题显示进度小点。

### 8.3 新建专题

- **用途**：创建专题骨架。
- **元素**：名称（必填）、密级（必选，下拉）、说明（选填）、初始素材（可选，可后补）。
- **交互**：确定后进入工作台并直接弹出汇入面板（§8.10）。
- **校验**：名称非空且专题内唯一；密级不得高于当前用户权限。

### 8.4 专题工作台 — 总览与导航

- **用途**：单个专题内的全部作业，核心界面。
- **布局**：顶部标签页 + 三栏主体。

```
┌─ 东南沿海动态 ────────────────────────[机密]──[● 离线]──[张三 ▾]──────────────┐
│ [素材] [要素] [问答] [报告] [审计]                                  [ + 汇入 ] │
├──────────────┬────────────────────────────────┬───────────────────────────────┤
│  左栏          │  中栏                            │  右栏                          │
│  素材库 +      │  阅读 / 校对区                    │  AI 分析助手                   │
│  要素导航      │  （当前选中素材的原文/转写/译文）  │  （带出处的问答）              │
└──────────────┴────────────────────────────────┴───────────────────────────────┘
```

- **标签页**：素材 / 要素 / 问答 / 报告 / 审计。切换改变中栏与右栏内容侧重。
- **右栏 AI 助手在所有标签页常驻**，上下文自动绑定当前专题。

### 8.5 工作台 · 素材

- **用途**：浏览、阅读、校对单件素材的加工结果。
- **布局**：

```
├──────────────┬────────────────────────────────┬───────────────────────────────┤
│ 素材库(23)    │  通话_0531.mp3   ▸ 转写 | 译文   │  AI 分析助手                   │
│ 📄 简报.docx  │  ───────────────────────────    │  ─────────────────            │
│ 🎙 通话.mp3 ◉ │  [00:12] 甲: 明天到货…           │  问: 涉及哪些人?               │
│ 🎬 监控.mp4   │  [00:31] 乙: 在老地方交接         │  答: 共识别 3 人                │
│ 🖼 照片.jpg   │  [01:05] 甲: 带上那批"货"         │   • 甲(说话人1) • 乙(说话人2)  │
│ + 汇入        │                                  │   • 老王(文件②提及)            │
│              │  [⚠ 00:31 置信度低，点此校对]     │   ⤷ 出处: 通话00:31 / 简报第2段│
│              │                                  │  [ 追问 ]  [ 加入报告 ]        │
└──────────────┴────────────────────────────────┴───────────────────────────────┘
```

- **左栏**：素材列表，图标区分类型；显示每件加工状态（待/中/完成/失败）。
- **中栏**：选中素材的内容；音视频显示带时间戳的转写，外文可切"原文/译文"；
  扫描件显示 OCR 文本（可对照原图）。低置信片段高亮 + "点此校对"。
- **交互**：点击转写时间戳可定位播放；校对入口见 §9.2。
- **状态**：加工中显示进度；失败显示原因 + 重试/手动录入。

### 8.6 工作台 · 要素

- **用途**：浏览自动抽取的人物/地点/组织/时间/装备/事件及其关系、时间线。
- **布局**：左=要素分类树；中=选中要素详情（别名、出现次数、全部提及列表，每条可点回原文）；
  右=关系网络/时间线视图。

```
├──────────────┬────────────────────────────────┬───────────────────────────────┤
│ 👤 人物 (12)  │  老王                            │  关系网络                      │
│ 📍 地点 (5)   │  别名: 老K / 王某                 │   甲 ──交接──→ 乙              │
│ 🏢 组织 (3)   │  出现 7 次                        │   乙 ──上线──→ 老王            │
│ ⚙ 装备 (2)   │  提及:                            │  ─────────────                │
│ 📅 事件 (9)   │   • 简报.docx 第2段「…老王负责…」 │  时间线                        │
│ 🕐 时间线 →   │   • 通话.mp3 00:31「老地方交接」   │   05-29 到货                   │
│              │   • 监控.mp4 02:14 画面识别        │   05-31 交接(老地方)           │
│              │   [全部 7 条 ▾]                   │   06-01 …                     │
└──────────────┴────────────────────────────────┴───────────────────────────────┘
```

- **交互**：点任一"提及"跳到素材对应位置（§9.4）；可人工合并/拆分要素、改别名、纠错
  （人在环路）。关系/时间线条目同样可溯源。
- **状态**：抽取中显示"正在抽取要素…"；空态提示先完成素材加工。

### 8.7 工作台 · 问答

- **用途**：对整个专题语料做自然语言问答与分析，结果带出处。
- **布局**：中栏=问答会话流；右栏可折叠为引用详情面板。

```
│  问: 这批材料里"那批货"指什么？有哪些佐证？                                    │
│  ──────────────────────────────────────────────────────────────────────────  │
│  答: 综合 3 处材料，"货"很可能指…（研判，非定论）                              │
│      佐证:                                                                     │
│       ①「带上那批"货"」— 通话_0531.mp3 00:31  [查看]                          │
│       ②「到货清单见附件」— 简报.docx 第3段     [查看]                         │
│       ③ 监控_0524.mp4 01:50 画面            [查看]                            │
│      [ 追问 ]   [ 加入报告 ]   [ 标记存疑 ]                                    │
```

- **交互**：每条引用可"查看"跳源；"加入报告"把结论+出处收进报告草稿；"标记存疑"
  让结论进入待核清单。区分**事实**与**研判**，研判明确标注。
- **约束**：无出处不作答——若语料不支持，回答"现有材料不足以判断"，不臆造。
- **状态**：检索/生成中显示进度；可中断。

### 8.8 工作台 · 报告

- **用途**：从专题成果生成公文式情报通报，人工复核后导出。
- **布局**：左=报告结构大纲（概要/小节/结论）；中=可编辑正文；右=可插入的素材/问答出处。

```
│  情报通报（草稿）                            状态: 草稿 → [ 提交复核 ]          │
│  ────────────────────────────────────────────────────────────────────────   │
│  标题:  关于东南沿海近期动态的情况通报                                          │
│  密级:  机密            签发: ____      日期: 2026-06-03                       │
│  概要:  [可编辑]                                                               │
│  一、…  [可编辑，右侧出处可一键插入引用]                                       │
│  二、…                                                                         │
│  结论/研判: [可编辑]                                                           │
│  ────────────────────────────────────────────────────────────────────────   │
│  [ 重新生成 ]   [ 保存草稿 ]   [ 提交复核 ]   (复核通过后) [ 导出 .md/.docx ]  │
```

- **交互**：一键生成草稿（复用 intel-bulletin 渲染）；正文可编辑；从右栏插入出处。
  **必须"提交复核"→ 复核人核准后才出现"导出"**（人在环路 + 审计）。
- **约束**：不替用户编造密级、文号、签发、日期等元数据，缺则留空并标"待核"。
- **状态**：草稿 / 待复核 / 已复核 / 已导出。

### 8.9 工作台 · 审计（专题内）

- **用途**：在专题内查看本专题相关的操作留痕（全局审计在管理后台 §8.15）。
- **元素**：时间、用户、动作（汇入/加工/问答/报告/导出）、对象、结果。只读、可筛选。

### 8.10 汇入与加工

- **用途**：导入素材并展示加工进度，给用户对"后台在做什么"的掌控感。
- **布局**：

```
┌─ 汇入素材 ────────────────────────────────────────────────────────────┐
│  拖入文件，或 [ 选择文件 ]      支持: 文档/音频/视频/图片，含多语种      │
│  ─────────────────────────────────────────────────────────────────   │
│  通话_0531.mp3    🎙 语音转写      ████████░░ 80%                       │
│  扫描件_01.pdf    📄 文字识别      ██████████ 完成 ✓                    │
│  监控片段.mp4     🎬 画面+语音     ███░░░░░░░ 30%                       │
│  外文简报.pdf     🌐 识别 + 翻译   ██████░░░░ 60%                       │
│  受损文件.dat     ✗ 无法识别 — [重试] [手动录入] [移除]                 │
│  ─────────────────────────────────────────────────────────────────   │
│  全部完成后将自动抽取要素、构建时间线、建立检索索引                      │
└─────────────────────────────────────────────────────────────────────────┘
```

- **交互**：拖拽/选择上传；逐件显示加工阶段与进度；失败项独立处理不阻塞其余。
- **状态**：每件 待加工 / 加工中 / 完成 / 失败；全部完成触发要素抽取与索引构建。

### 8.11 管理后台 · 提示词模板

- **用途**：管理 AI 各任务用的提示词模板（系统提示、要素抽取、问答、报告等），
  可编辑、版本化、启停，**无需改代码**。
- **元素**：模板列表（名称/适用场景/版本/启用）；编辑器（支持变量占位）；版本历史与回滚。
- **交互**：新建/编辑/复制/启停/回滚；保存即新版本。变更入审计。
- **约束**：内置基线模板只读，自定义在其副本上进行。

### 8.12 管理后台 · Skill 管理

- **用途**：管理可插拔技能（情报通报生成、音视频对话分析、关系网络抽取等）。
- **布局**：

```
┌─ Skill 管理 ───────────────────────────────────────────────────────────┐
│  情报通报生成    v2.0   ● 启用    [配置] [测试] [评测结果]               │
│  音视频对话分析  v1.0   ● 启用    [配置] [测试]                         │
│  关系网络抽取    v0.5   ○ 停用    [配置] [测试]                         │
│  ─────────────────────────────────────────────────────────────────   │
│                                                  [ + 导入 Skill ]      │
└─────────────────────────────────────────────────────────────────────────┘
```

- **元素**：列表（名称/版本/启停/描述）；配置面板；"评测结果"展示该 skill 的 evals 通过情况。
- **交互**：启停、配置参数、导入新 skill（离线包）、跑测试/评测。直接对接既有 skill 系统。

### 8.13 管理后台 · 模型配置

- **用途**：配置各用途的开源模型接入端点与"算力档位"。屏蔽业务用户，仅管理员可见。
- **元素**：分用途配置——文本 / 多模态(图像视频) / 语音(ASR) / 嵌入检索；每项含
  provider、模型名、Base URL、密钥（脱敏显示）、连通状态（在线/离线/未配置）、超时。
  顶部"算力档位"切换（开发/单卡/服务器）一键套用一组默认。
- **交互**：编辑+测试连通（"自检"按钮，对应既有 `doctor`）；切换档位。
- **约束**：界面提示"仅限开源模型"；保存的密钥不回显明文；配置变更入审计。

### 8.14 管理后台 · 用户与权限

- **用途**：管理本地账号、角色、可访问密级。
- **元素**：用户列表（姓名/角色/可访问密级/启停）；新增/编辑/停用；重置口令。
- **约束**：用户只能进入不高于自身密级的专题；删除用户保留其审计记录。

### 8.15 管理后台 · 审计中心

- **用途**：保密员查看全量、防篡改的审计日志，复核边界事件并导出留存。
- **布局**：

```
┌─ 审计中心 ──────────────────────────────[完整性校验: ✓ 链未断]──────────┐
│  筛选: [用户 ▾] [动作 ▾] [专题 ▾] [时间段]                  [ 导出 ]    │
│  ─────────────────────────────────────────────────────────────────   │
│  时间              用户    动作        对象             结果            │
│  06-03 14:02:11   张三    汇入素材    通话_0531.mp3    成功            │
│  06-03 14:05:30   张三    问答        "那批货指什么"    成功            │
│  06-03 14:20:08   李四    报告导出    东南沿海动态.docx 成功            │
│  06-03 14:21:55   系统    外发拦截    127.0.0.1✗外联    已阻断 ⚠       │
└─────────────────────────────────────────────────────────────────────────┘
```

- **元素**：可筛选事件表；顶部哈希链完整性校验状态；"外发拦截"类事件高亮告警。
- **交互**：筛选、查看详情、导出留存（导出本身也是一条审计）。只读，不可改业务数据。

## 9. 关键交互模式

### 9.1 溯源引用（贯穿全产品）

AI 的每条结论、每个要素、每段研判都携带**出处列表**：`素材 + 位置(页/段/时间戳) + 原文片段`。
出处可点击跳转到素材对应位置并高亮（§9.4）。无出处不下结论。

### 9.2 人工校对（OCR / ASR）

低置信片段以高亮 + "⚠ 点此校对"标记。点击进入校对态：左原文/原图，右可编辑文本，
保存后该片段重算关联要素。校对动作入审计。

### 9.3 报告复核闸门

报告导出强制经过"草稿 → 提交复核 → 复核人核准 → 导出"。未复核不可导出。
体现"AI 是参谋、人是决策者"。

### 9.4 要素↔素材双向跳转

点要素→列出全部提及，点提及→定位素材；反向在素材中选中文本→"标为要素/关联已有要素"。

### 9.5 事实与研判分离

界面与文案明确区分"材料中的事实"与"AI 的研判/推测"，研判加标识，避免被当成定论。

## 11. 安全 · 离线 · 审计在产品层的体现

### 11.1 密级可见

顶栏常驻密级徽标，分级配色；专题、报告、导出文件均带密级标识。

### 11.2 离线状态

顶栏"● 离线"常驻；系统不提供任何主动外发功能；检测到外发尝试即转告警态。

### 11.3 审计哈希链

每条审计事件含 `本条哈希 = H(内容 + 上一条哈希)`，形成链，事后无法静默篡改；
审计中心顶部显示"链完整性校验"。覆盖：登录、汇入、加工、校对、问答、报告生成/复核/导出、
配置变更、外发拦截。可关联运行时 trace 作为补充诊断证据，但业务审计以独立 AuditEvent 为准。

## 13. 工程前置约束与一期边界

> 本节不改变 §1–§12 的产品方向与逐屏结构，只为工程落地划定前置门槛与一期边界，
> 避免"界面先行、底座落空"。

### 13.1 离线媒体链路是阶段一前置门槛

现有 `av-dialogue-insight` 的音视频路径依赖**豆包(Doubao) ASR、DashScope 云多模态、
火山 TOS / 公网 URL**，与本产品"开源 + 不外发"硬约束直接冲突。阶段一**必须先禁用、替换或
隔离这些路径**，否则不得进入正式产品链路：

- 涉密 / 生产环境：上述云路径一律禁用，仅接入开源模型与本地落盘。
- 非涉密开发环境：允许用云端开源模型替身加速开发，但**必须在界面与配置中显式标记"开发模式"**
  （顶栏徽标 + 模型配置页警示），且开发模式下不得接入任何涉密专题。
- 门槛未达成前，§8.5 / §8.10 的转写、识别能力按"该能力暂不可用"降级（§10）。

### 13.2 溯源必须先定义统一 Citation 数据契约

§9.1 的"溯源"在工程上落为一个**统一的 Citation 契约**，被所有 AI 输出、要素、关系、
时间线、报告段落共同引用：

```jsonc
{
  "material_id":   "...",            // 素材唯一标识
  "material_name": "...",            // 显示名
  "modality":      "doc|audio|video|image",
  "locator":       { "page?": 0, "paragraph?": 0, "timecode?": "00:31", "bbox?": [0,0,0,0] },
  "snippet":       "...",            // 命中的原文片段
  "derived_from":  ["citation..."],  // 可选：由哪些更底层出处推导而来
  "confidence":    0.0,              // 0–1
  "content_hash":  "..."             // 指向素材的内容哈希，素材被改后引用即失效告警
}
```

**无 citation 的内容只能作为"待核提示"呈现，不能作为事实或写入报告结论**（与 §8.7、§9.5 一致）。

### 13.3 审计不是运行 trace 的简单复用

§11.3 的审计与运行时既有 trace **是两个东西**：trace 是**运行诊断**（面向调试），
AuditEvent 是**业务留痕**（面向保密复核）。AuditEvent 必须独立定义为：

- **append-only**，不可改写、不可删除；
- **哈希链**防篡改（§11.3），可独立校验；
- **可导出**留存、**可校验**完整性。

trace 可作为审计的**补充证据**被关联引用，但**不能替代**审计中心（§8.15）。

### 13.4 工作区统一以 Case 为产品边界

UI 与产品层**只暴露 `cases/<case-id>/`**（§5.3）。现有 `tasks/<id>/`、`av-tasks/<id>/`
仅作为 skill 内部兼容层或一次性迁移输入，**不应长期与 Case 并列暴露给用户**；迁移后旧目录
对用户不可见。

### 13.5 一期范围要降风险（MVP）

一期把"能跑通、可验收"的最小闭环做扎实，其余先占位：

| 一期优先做（做实） | 先做只读 / 占位 / 最小可用 |
| --- | --- |
| 专题 CRUD | 提示词版本管理（先内置基线、不可编辑） |
| 素材汇入 + 加工状态 | Skill 离线导入（先固定内置 skill） |
| 统一落盘（Case 化） | 模型档位（先单一配置，不做档位切换） |
| 报告草稿 / 复核 / 导出闸门 | 用户权限（先单角色或最简登录） |
| 最小审计链（append-only + 哈希链 + 校验） | 报告多级会签（先单级复核） |
| 管理后台骨架（可进入、占位可见） | 复杂关系图（先列表 + 简单连线） |

## 14. 非目标 / 暂不做

- 网络爬取 / 主动采集（数据只进不出，仅分析已提供文件）。
- 模型选型、部署、量化、组网（由部署方负责，本系统视模型为可插拔接入）。
- 版式精排（报告仅保证公文结构规范）。
- 跨专题的全库联合检索（一期以专题内为界，可作后续演进）。

## 15. 术语表

| 术语 | 含义 |
| --- | --- |
| 专题(Case) | 一个分析任务的工作单元，含其全部素材与分析成果 |
| 素材(Material) | 汇入的单个文件及其加工结果 |
| 要素(Element) | 从素材中抽取的人/地/组织/时间/装备/事件等结构化条目 |
| 溯源 | 任一结论可回链到原始素材的出处 |
| 加工 | OCR / 转写 / 翻译 / 要素抽取 / 索引构建的统称 |
| 算力档位 | 按目标硬件套用的一组模型配置（开发/单卡/服务器） |

## 16. 待定问题

1. **算力档位**的具体硬件边界与默认模型，待部署硬件明确后填充（不影响界面设计）。
2. **多用户/口令/二要素**的强度要求，依实际密级与等保要求确定。
3. **跨专题检索**是否纳入二期。
4. **报告复核**是否需要多级会签。
5. **要素关系图**的可视化深度（一期可先做列表 + 简单连线，复杂图谱后置）。
6. **一期硬性准入标准**：是否以"默认无外发、所有结论有 citation、报告未复核不可导出、
   审计链可校验"四条作为一期的硬性验收准入（建议采纳，对应 §13）。
